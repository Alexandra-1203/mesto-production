Mesto — это интерактивное веб-приложение, аналог мини-социальной сети для путешественников для обмена карточками с изображениями. Пользователь может:
*   Редактировать свой профиль (имя, род занятий).
*   Добавлять новые карточки с названием и ссылкой на изображение.
*   Ставить и снимать лайки понравившимся карточкам.
*   Удалять собственные карточки.
*   Открывать фотографии в полноразмерном режиме.
*   Все изменения сохраняются на сервере.
*   Посмотреть статистику карточек при нажатии на лого.


🔗 Посмотреть проект: https://alexandra-1203.github.io/mesto-production/


Установка и запуск:

git clone https://github.com/Alexandra-1203/mesto-add.git

npm install

npm run dev

npm run build


Чему я научилась в этом проекте

1. Продвинутая работа с DOM и событиями
   
*   Управление состоянием интерфейса через JavaScript.
*   Создание и удаление HTML-элементов (карточек) динамически.
*   Обработка множества событий: клик, сабмит форм, нажатие клавиш (Esc для закрытия попапов).
*   Использование делегирования событий для эффективной обработки лайков и удаления карточек.

2. Асинхронность и работа с API
*   Взаимодействие с сервером через `fetch` (GET, POST, PATCH, PUT, DELETE запросы).
*   Загрузка данных пользователя и начального набора карточек с сервера.
*   Отправка обновленных данных профиля и новых карточек на сервер.
*   Обработка ответов от сервера и отлов ошибок с помощью `catch`.

3. Валидация форм
*   Написание собственной логики валидации без использования библиотек.
*   Кастомные сообщения об ошибках под каждым полем ввода.
*   Блокировка кнопки отправки, пока форма невалидна.
*   Очистка ошибок при закрытии попапа.

4. Модальные окна (Попапы)
*   Реализация плавного открытия и закрытия всех форм (редактирование профиля, добавление карточки, просмотр фото, подтверждение удаления, обновление аватара).
*   Универсальная логика для работы с разными типами попапов.
*   Закрытие попапа по клику на оверлей и по нажатию на `Escape`.

5. ООП и модульная структура
*   Организация кода с использованием классов (`Card`, `FormValidator`, `Popup`, `UserInfo`, `Api`).
*   Разделение кода на модули (JavaScript-файлы для каждой логической части).
*   Сборка проекта через Vite.

6. Улучшение UX
*   Отображение состояния загрузки (изменение текста на кнопках на "Сохранение...").
*   Плавные анимации открытия/закрытия попапов.
*   Мгновенное обновление интерфейса после успешного запроса к серверу.

7. Углубленная верстка и организация кода
*   Grid Layout для сетки карточек.
*   Адаптивная верстка под разные разрешения экранов.
*   Организация стилей по методологии БЭМ.
*   CSS-переменные для цветовой схемы.


Технологии

*   HTML5, CSS3 (Grid, Flexbox, адаптивность, БЭМ)
*   JavaScript (ES6+): классы, модули, промисы, асинхронные функции
*   Vite: сборка проекта, минификация, обработка ресурсов
*   Git: работа с ветками, публикация на GitHub Pages
*   REST API: взаимодействие с бэкендом
*   ООП: проектирование классов и их взаимодействия
